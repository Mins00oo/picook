## 2. 서비스별 구성

### 2.1 AWS EC2 / Lightsail (메인 서버)
| 항목 | 설정 |
|------|------|
| 인스턴스 | Lightsail $5~10/월 (1GB~2GB RAM) 또는 EC2 t3.small |
| OS | Ubuntu 24.04 LTS |
| 런타임 | Java 21, Docker |
| 배포 | Git pull → Gradle build → systemd 재시작 |
| 도메인 | api.picook.com (Phase 2) |
| SSL | Let's Encrypt (Certbot) |
| 리버스 프록시 | Nginx → Spring Boot :8080 |

**주의**: 쇼츠 변환 시 ffmpeg 처리로 CPU/메모리 사용 높음 → 최소 2GB RAM 권장.
서버에 **yt-dlp**, **ffmpeg** 사전 설치 필요.

### 2.2 Docker PostgreSQL 15
| 항목 | 설정 |
|------|------|
| 이미지 | postgres:15 |
| 로컬 | localhost:5432 |
| 프로덕션 | EC2 내 Docker (같은 서버) |
| DB/User | picook_db / picook_user |
| 볼륨 | picook_pgdata (데이터 영속화) |
| 백업 | cron + pg_dump → S3 업로드 (일 1회) |

### 2.3 AWS S3 (스토리지)
| 항목 | 설정 |
|------|------|
| 버킷 | picook-bucket |
| 리전 | ap-northeast-2 (서울) |
| 용도 | 레시피 이미지, 재료 아이콘, 프로필 사진, 완성 사진 |
| 접근 | Spring Boot에서 AWS SDK v2로 업로드, 프리사인 URL로 읽기 |
| 비용 | S3 Standard, 5GB까지 무료 티어 |

### 2.4 Vercel (백오피스)
Hobby(무료), Vite+React Static, GitHub 자동 배포, SSL 자동.

### 2.5 Expo EAS (모바일)
Free(30빌드/월), EAS Build, EAS Submit, EAS Update(OTA).

## 4. CI/CD

모노레포 → **변경 디렉토리 감지** → 해당 파이프라인만 실행.

### 4.1 백엔드 (backend/)
```
GitHub push → paths-filter(backend/**) → Gradle build+test → SSH EC2 → git pull → build → restart
```

### 4.2 모바일 (mobile/)
```
GitHub push → paths-filter(mobile/**) → npm ci + tsc + jest → (수동) eas build
```

### 4.3 백오피스 (admin/)
```
GitHub push → Vercel 자동 배포
```

## 9. 백업 및 복구

| 대상 | 방식 | 주기 |
|------|------|------|
| PostgreSQL | pg_dump → gzip → S3 업로드 | 일 1회 (cron) |
| S3 | 자체 내구성 (99.999999999%) | — |
| 코드 | GitHub | 커밋 시 |

```bash
# infra/scripts/backup.sh
pg_dump -h localhost -U picook_user picook_db | gzip > /tmp/picook_$(date +%Y%m%d).sql.gz
aws s3 cp /tmp/picook_$(date +%Y%m%d).sql.gz s3://picook-backup/db/
```

복구: S3에서 다운 → pg_restore. 앱 긴급 패치: EAS Update OTA (30분).
